## Общая информация

### Архитектура

Система состоит из двух основных компонентов:

1. **auth.js** - Класс `AuthSystem` управляет:
   - Регистрацией пользователей
   - Входом/выходом
   - Профилями пользователей
   - Хешированием паролей
   - Отслеживанием сессий

2. **cabinet.html** - Интерфейс личного кабинета:
   - Формы регистрации и входа
   - Профиль пользователя
   - История заказов
   - Настройки безопасности

### Хранилище данных

Все данные хранятся в `localStorage`:

```javascript
localStorage.getItem('system_users')     // Все зарегистрированные пользователи
localStorage.getItem('current_user')    // Текущий залогиненный пользователь
localStorage.getItem('tracking_sessions')// Отслеживание входов
localStorage.getItem('real_orders')     // История заказов
localStorage.getItem('action_logs')     // Логи действий
```

## Регистрация

### Форма регистрации

Поля обязательны:
- **ФИО** - Полное имя пользователя (2+ символа)
- **Email** - Действительный email (уникальный)
- **Пароль** - Минимум 6 символов
- **Телефон** - Опционально

### Процесс регистрации

1. Заполните все поля
2. Нажмите "Зарегистрироваться"
3. Система проверит:
   - Есть ли пользователь с таким email
   - Длину пароля (минимум 6 символов)
   - Наличие всех обязательных полей
4. При успехе - подтверждение, переходите к входу
5. При ошибке - сообщение об ошибке под формой

### Пример успешной регистрации

```
Имя: Алексей Петров
Email: alex.petrov@email.com
Пароль: MySecure123
Телефон: +7 (995) 432-10-98

Результат: ✓ Регистрация успешна!
          Теперь вы можете войти
```

### Возможные ошибки

| Ошибка | Причина | Решение |
|--------|---------|---------|
| "Все поля обязательны" | Не заполнены обязательные поля | Заполните Email, Пароль и ФИО |
| "Пароль должен быть минимум 6 символов" | Пароль слишком короткий | Введите пароль минимум 6 символов |
| "Пользователь с этим email уже существует" | Email уже зарегистрирован | Используйте другой email или войдите |

## Вход в систему

### Форма входа

Требуемые поля:
- **Email** - Email зарегистрированного пользователя
- **Пароль** - Пароль, используемый при регистрации

### Процесс входа

1. Введите email и пароль
2. Нажмите "Войти"
3. Система проверит:
   - Существование пользователя
   - Правильность пароля
   - Статус аккаунта (активен/заблокирован)
4. При успехе - перейдёте в кабинет
5. При ошибке - сообщение об ошибке

### Тестовые аккаунты

**Примечание**: При первом использовании нет предзаполненных аккаунтов. Сначала создайте свой.

### Пример успешного входа

```
Email: alex.petrov@email.com
Пароль: MySecure123

Результат: ✓ Вход успешен!
Перенаправление в кабинет...
```

### Возможные ошибки

| Ошибка | Причина | Решение |
|--------|---------|---------|
| "Email и пароль обязательны" | Не заполнены поля | Заполните оба поля |
| "Пользователь не найден" | Email не зарегистрирован | Сначала пройдите регистрацию |
| "Неправильный пароль" | Введён неправильный пароль | Проверьте пароль (чувствительны к регистру) |
| "Аккаунт заблокирован" | Аккаунт администратором заблокирован | Свяжитесь с поддержкой |

## Управление профилем

### Редактирование профиля

После входа в кабинет:

1. Нажмите кнопку "Редактировать" рядом с вашим именем
2. В открывшейся вкладке "Профиль" можете изменить:
   - **Телефон** - Контактный номер
   - **Компания** - Название вашей организации
   - **Адрес** - Полный адрес доставки
   - **ИНН** - Налоговый идентификационный номер
   - **КПП** - Код причины постановки на учет
   - **БИК** - Банковский идентификационный код

3. Нажмите "Сохранить изменения"
4. Получите подтверждение

### Информация в профиле

Система автоматически сохраняет:
- Дата регистрации
- Статус аккаунта
- Количество заказов
- ID пользователя

### Пример редактирования

```
Исходные данные:
- Email: ivan@example.com
- ФИО: Иван Петров
- Телефон: (пусто)

Новые данные:
- Телефон: +7 (999) 123-45-67
- Компания: ООО МойТранспорт
- Адрес: Москва, ул. Ленина, 1

Результат: ✓ Профиль обновлён
```

## История заказов

### Просмотр заказов

1. В кабинете перейдите на вкладку "Мои заказы"
2. Система отобразит все ваши заказы

### Информация о заказе

Каждый заказ содержит:
- **ID заказа** - Уникальный номер (ТН-XXXXXX)
- **Дата создания** - Когда был создан заказ
- **Маршрут** - Город отправления → Город назначения
- **Стоимость** - Сумма в рублях
- **Тип груза** - Категория груза (Общий, Хрупкий и т.д.)
- **Вес и объем** - Параметры груза
- **Статус** - Текущее состояние заказа

### Возможные статусы заказа

| Статус | Описание | Цвет |
|--------|---------|------|
| На рассмотрении | Заказ создан, ожидает подтверждения | 🟡 Оранжевый |
| Отправлен | Заказ принят в обработку | 🔵 Голубой |
| В пути | Груз находится в пути | 🟢 Зелёный |
| Завершен | Груз доставлен | ✅ Зелёный |
| Отменен | Заказ отменён | ❌ Красный |

### Создание заказа

1. Перейдите на **calculator.html**
2. Заполните детали заказа:
   - Выберите города
   - Укажите вес и объём
   - Выберите тип груза
3. Нажмите "Рассчитать и добавить в корзину"
4. Перейдите в **cart.html**
5. Нажмите "Оформить заявку"
6. Заказ будет добавлен в историю

**Примечание**: Заказы автоматически привязываются к текущему пользователю.

## Отслеживание сессий

### Как это работает

1. **Автоматическое отслеживание**: При входе создаётся сессия
2. **Сбор данных**:
   - ID сессии
   - Дата и время входа
   - IP адрес (если доступен)
   - Информация об устройстве
   - Страницы посещения
   - Все действия пользователя

3. **Хранение**: Данные сохраняются в `localStorage`
4. **Доступ**: Доступно в админ-панели (`admin.html`)

- ✅ Вход/выход пользователя

- ✅ Создание заказов
- ✅ Отправка заказов
- ✅ Редактирование профиля

## Безопасность

### Текущее состояние

⚠️ **Это тестовое приложение**. Система использует простое хеширование паролей.

### Хеширование паролей

Пароли хешируются простой функцией:

```javascript
hashPassword(password) {
    let hash = 0;
    for (let i = 0; i < password.length; i++) {
        const char = password.charCodeAt(i);
        hash = ((hash << 5) - hash) + char;
        hash = hash & hash;
    }
    return 'hash_' + Math.abs(hash);
}
```

**Примечание**: Это НЕ криптографически безопасно! Используется только для тестирования.

### Рекомендации для production

1. **Серверная валидация** - Все проверки должны быть на сервере
2. **Криптографическое хеширование** - Используйте bcrypt или argon2
3. **SSL/HTTPS** - Все данные передаются по защищённому каналу
4. **Токены доступа** - Используйте JWT или OAuth
5. **Двухфакторная аутентификация** - Добавить 2FA
6. **Rate limiting** - Ограничить попытки входа
7. **CSRF protection** - Защита от атак
8. **Logging** - Логировать все попытки входа на сервере

### Лучшие практики

- ✅ **Используйте уникальные пароли** для каждого сервиса
- ✅ **Не делитесь паролем** с никем
- ✅ **Выходите** при использовании публичных компьютеров
- ✅ **Проверяйте URL** перед входом
- ⚠️ **Не вводите пароль** в небезопасных местах

## FAQ

### Q: Забыл пароль, что делать?

**A**: В этой версии нет функции восстановления пароля. Создайте новый аккаунт с другим email. В production нужна функция "Забыли пароль?"

### Q: Может ли администратор видеть мой пароль?

**A**: Нет, пароли хешируются. В production используется криптографическое хеширование, пароли не видны даже администраторам.

### Q: Как долго мой аккаунт остаётся активным?

**A**: Аккаунт остаётся активным неограниченно. В production можно добавить сроки действия.

### Q: Что произойдёт с моими данными, если я удалю аккаунт?

**A**: Аккаунт и все данные удаляются. Это действие необратимо. Кнопка в разделе "Безопасность".

### Q: Где хранятся мои данные?

**A**: В `localStorage` браузера. При очистке истории браузера данные удалятся. В production - на защищённом сервере.

### Q: Безопасно ли использовать это на мобильном устройстве?

**A**: Да, система полностью адаптивна для мобильных. Все функции работают одинаково.
