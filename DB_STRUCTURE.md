# 🚛 База данных ТрансНафта — Готово к использованию!

## ✅ Что создано

Я разработал **полнофункциональную базу данных SQLite** для вашего логистического сайта **ТрансНафта**. База работает с инструментом **DBBrowser for SQLite** и готова к использованию.

### 📦 Созданные файлы

| Файл | Размер | Описание |
|------|--------|---------|
| **`database.sql`** | ~30 KB | SQL-схема всех таблиц, индексов и триггеров |
| **`init_database.py`** | ~8 KB | Python скрипт для создания и управления БД |
| **`db.js`** | ~12 KB | Node.js модуль с готовыми методами для работы с БД |
| **`DB_SETUP_GUIDE.md`** | ~20 KB | Полное руководство по установке и использованию |
| **`DATABASE_GUIDE.md`** | ~40 KB | Описание всех таблиц, полей и структуры |
| **`SQL_QUERIES_AND_API.md`** | ~25 KB | 50+ примеров SQL запросов и API примеров |
| **`DB_STRUCTURE.md`** | - | Этот файл |

## 🗄️ Структура базы данных

### 15 основных таблиц

```
📊 БИЗНЕС-ДАННЫЕ
├── users              → 👤 Пользователи, клиенты, водители, администраторы
├── orders             → 🛒 Заказы доставки (основная таблица)
├── services           → 📦 Услуги (авто, контейнер, рефрижератор, экспресс, склад)
├── drivers            → 👨‍✈️ Водители с лицензиями и статистикой
├── vehicles           → 🚗 Транспортные средства (грузовики, контейнеры)
├── delivery_regions   → 🗺️ Регионы доставки с расценками

💳 ФИНАНСЫ И ПЛАТЕЖИ
├── payments           → 💳 Все платежи по заказам
├── promo_codes        → 🎁 Промокоды и скидки
├── promo_usage        → 📝 История использования промокодов

⭐ КЛИЕНТСКАЯ ОБРАТНАЯ СВЯЗЬ
├── reviews            → ⭐ Отзывы и рейтинги клиентов

📍 ЛОГИСТИКА И ОТСЛЕЖИВАНИЕ
├── order_tracking     → 🗺️ GPS точки отслеживания грузов
├── order_statuses     → 📊 Справочник статусов заказов

🆘 СЛУЖБА ПОДДЕРЖКИ
├── support_tickets    → 🎫 Тикеты поддержки
├── ticket_messages    → 💬 Сообщения в тикетах

📈 АНАЛИТИКА И ЛОГИРОВАНИЕ
├── analytics          → 📊 Ежедневная статистика и метрики
└── activity_logs      → 📋 Логи всех действий в системе
```

### 4 представления (Views) для удобного анализа

- **`order_summary`** — сводная информация по всем заказам с деталями клиента и водителя
- **`driver_stats`** — статистика по водителям (рейтинги, количество рейсов, доход)
- **`monthly_stats`** — ежемесячная статистика (заказы, доход, рейтинги)
- **`pending_support_tickets`** — активные тикеты поддержки, отсортированные по приоритету

### 17 индексов для оптимизации

Созданы индексы на всех часто используемых полях для быстрого поиска:
- Индексы на `email`, `status`, `user_id`, `order_id`
- Временные индексы для фильтрации по датам
- Индексы на коды (`payment_method`, `service_type` и т.д.)

### 5 триггеров для автоматизации

- Автоматическое обновление `updated_at` при изменении записей
- Увеличение счётчика заказов пользователя
- Увеличение счётчика использования промокода
- И другие автоматизированные операции

2. **File → Open Database**

4. ✅ Готово! Можно работать с БД

## 🔧 Интеграция с вашим сайтом

### С JavaScript/Node.js

```javascript
const db = require('./db');

// Регистрация клиента
const userId = await db.createUser({
    email: 'ivan@example.ru',
    password: 'secret123',
    fullName: 'Иван Иванов',
    phone: '+7 999 123-45-67'
});

// Создание заказа
const order = await db.createOrder({
    userId: userId,
    serviceId: 1,
    cargoWeight: 5.5,
    cargoDescription: 'Запчасти',
    // ... остальные параметры
});

// Получение статистики
const stats = await db.getDailyStats();
```

### С Python

```python
import sqlite3

conn = sqlite3.connect('transnafta.db')
cursor = conn.cursor()

# Получить заказы пользователя
cursor.execute('SELECT * FROM orders WHERE user_id = ?', (1,))
orders = cursor.fetchall()

# Добавить отзыв
cursor.execute('''
    INSERT INTO reviews (order_id, user_id, rating, comment)
    VALUES (?, ?, ?, ?)
''', (1, 1, 5, 'Отличная доставка!'))

conn.commit()
```


## 🆘 Возникли проблемы?

### БД не создаётся
```bash
# Проверьте, что файл database.sql существует
ls database.sql

# Если нет, скачайте заново
# Если да, попробуйте создать вручную:
sqlite3 transnafta.db < database.sql
```

### Python не найден
```bash
# Проверьте версию
python --version

# Должно быть 3.7 или выше
# Если нет, установите Python с python.org
```

### DBBrowser не открывает файл
```bash
# Убедитесь, что файл существует
ls -la transnafta.db

# Проверьте размер (должен быть > 50 KB)
# Если меньше, БД не инициализирована
```

### БД работает медленно
- Откройте в DBBrowser → **Edit → Pragmas**
- Установите `cache_size = -65536`
- Установите `journal_mode = WAL`

Подробнее см. **DB_SETUP_GUIDE.md** раздел "Решение проблем"

## 📞 Справочная информация

### Типы пользователей
- `customer` — клиент (может делать заказы)
- `driver` — водитель (может принимать заказы)
- `admin` — администратор (полный доступ)

### Статусы заказов
- `pending` — ожидание подтверждения
- `confirmed` — подтверждён, назначен водитель
- `in_progress` — в пути
- `delivered` — доставлен
- `cancelled` — отменён
- `disputed` — спор

### Методы оплаты
- `card` — кредитная карта
- `bank_transfer` — банковский перевод
- `cash` — наличные
- `invoice` — счёт-фактура

### Типы услуг
- `auto` — автомобильные перевозки
- `container` — контейнерные перевозки
- `refrigerator` — рефрижераторные
- `express` — экспресс-доставка
- `warehouse` — складские операции
